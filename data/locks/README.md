# data/locks

## Purpose

Planned SHA-256 locks for raw files, Gematria profiles, transcript versions, and normalized corpus outputs.

## What belongs here

Future lock manifests with paths, hashes, sizes, source metadata, and profile IDs.

## What does not belong here

Raw files themselves, generated search outputs, or databases.

## Codex modification policy

Codex may edit documentation. Lock files should be generated only after Stage 0B source policy is defined.

## Stage 0A restrictions

No corpus locks are created in Stage 0A.
