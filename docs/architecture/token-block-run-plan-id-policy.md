# Token-Block Run-Plan ID Policy

Stage 5BD run-plan IDs are deterministic identifiers for future review of dry-run plan metadata. They are not byte-stream hashes and do not prove that a token experiment ran.

The policy uses canonical JSON serialization of metadata-only plan fields followed by SHA-256, with a stable 16-character prefix in the `stage5bd-...` ID. The digest excludes generated bytes, absolute paths, timestamps, raw image data, review-pack bodies, and local machine-specific state.

Run-plan IDs currently cover representative plan families only. Stage 5BD records the full family count separately and explicitly does not enumerate the Cartesian product.

Required invariants:

- `execution_authorised=false`
- `run_plan_ids_include_generated_bytes=false`
- `full_plan_family_cross_product_enumerated=false`
- `real_token_block_byte_streams_generated=false`
- `variant_branches_materialised=false`
