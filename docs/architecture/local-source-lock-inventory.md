# Local Source Lock Inventory

Stage 5AG inventories user-provided material already placed under ignored local roots, primarily `third_party/`. The inventory is metadata-only: file counts, sizes, SHA-256 hashes, extension summaries, archive member listings, duplicate hash groups, manifest linkage, source-lock candidate status, and research-bundle readiness.

This layer does not make local files canonical. Archive member listings are convenience metadata, not source truth. Raw source bytes, extracted bodies, generated full inventories, images, PDFs, HTML, DOCX, audio, video, archives, databases, and research bundles remain ignored and uncommitted.

The Stage 5AG source-lock path is:

1. Inventory ignored local files and archives.
2. Link local paths to Stage 5AF manifest source IDs where possible.
3. Record missing, manual-export, unsupported, duplicate, and unclassified material explicitly.
4. Produce compact source-lock candidate and research-bundle readiness metadata.
5. Defer curated extraction to Stage 5AH.

Network fetching, online cloning, browser automation, Google Drive project storage, Deep Research, hypothesis generation, CUDA, benchmarks, scored experiments, website expansion, canonical corpus activation, page-boundary finalisation, and solve claims are outside Stage 5AG.
