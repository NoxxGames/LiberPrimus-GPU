# Scraper Capture Profiles

Stage 5AJ adds scraper-capture policy for future source-harvester work. Profiles describe what a future bounded capture should preserve, such as canonical URL, retrieval timestamp, source class, HTTP metadata where available, page title, visible text, outbound links, hashes, and warnings.

These profiles do not authorize live scraping. Network fetches, online repository cloning, broad forum crawling, Google Drive storage, OCR, AI/ML interpretation, and website expansion remain out of scope unless a later prompt explicitly enables a bounded local policy.

For private Deep Research, scraper-derived extracts must preserve technical text and link context. For public website views, only reviewed, privacy-safe summaries may be exposed.
