# Token-Block Branch Eligibility Policy

Stage 5AY classifies Stage 5AW unresolved variant options before any future bounded preflight can be implemented.

Eligibility records distinguish canonical current tokens, generated candidates, reviewer-extra possible tokens, visual placeholders, malformed fragments, primary-60 mappable options, primary-60 unmappable options, and execution-ineligible options.

Important boundaries:

- Canonical current tokens remain the default baseline.
- Confirmed keep-current decisions confirm the baseline but do not rewrite it.
- Unresolved options remain unresolved.
- Primary-60 mappable options may be planned for future byte-stream variants only after later authorization.
- Visual placeholders such as `?4` and `3?` are preserved for review but are not byte-stream eligible.
- Malformed prose fragments are audit-only and never variant choices.
- Full Cartesian enumeration remains blocked.
- Sampling is design-only and requires a fixed seed plus later authorization.
