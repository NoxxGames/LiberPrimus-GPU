# Development Logs

## Purpose

Chronological engineering logs for ingestion and validation stages.

## What belongs here

Concise Markdown logs describing starting state, implementation choices, validation results, Git safety, and known limitations.

## What does not belong here

Secrets, raw data, generated JSON/JSONL outputs, downloaded files, logs from tools, or large dumps.

## Codex modification policy

Codex must create or update a development log for every ingestion stage.

## Stage 0C restrictions

Logs must describe non-canonical source handling and avoid solve claims.
