# Stage 4I Scorer Consolidation And Calibration

Stage 4I preserves Stage 3C scoring calibration as durable project state and connects it to the Stage 4H CPU batch API.

Research boundaries:

- scoring is triage only
- confidence labels cannot imply solved plaintext
- noisy and negative/deprioritised families remain documented
- no new scorer, model, broad campaign, CUDA work, or solve claim is introduced
